# Multi-language (i18next)

This app uses **i18next** + **react-i18next** with JSON locale files.

## Languages

| Code | Language | Default |
|------|----------|---------|
| `en` | English  | Yes     |
| `hi` | Hindi    |         |
| `gu` | Gujarati |         |

## File structure

```
public/locales/           ← edit here (canonical for content authors)
  en/translation.json
  hi/translation.json
  gu/translation.json
src/i18n/
  i18n.js                 ← i18next init (localStorage persistence)
  locales/                ← bundled copy (synced from public/)
src/context/I18nContext.js
src/components/LangSwitcher.js
```

After editing `public/locales/**/*.json`, sync the bundle:

```bash
node scripts/build-locales.mjs
```

Or edit `src/i18n/locales/` directly during development.

## Default & persistence

- **Default:** English on first visit
- **Persistence:** `localStorage` key `drj_lang_v1`
- **Switcher:** header `LangSwitcher` (EN / हिं / ગુ)

## Replace hardcoded strings

### Existing components (nested object)

```jsx
import { useI18n } from "@/context/I18nContext";

const { t } = useI18n();
return <h1>{t.shop.title}</h1>;
```

### New code (`t()` function — recommended)

```jsx
import { useTranslation } from "react-i18next";

const { t } = useTranslation();
return <h1>{t("shop.title")}</h1>;
```

Add keys to JSON:

```json
{
  "shop": {
    "title": "Shop"
  }
}
```

## App wiring

1. `src/index.js` — `import "@/i18n/i18n"` before `<App />`
2. `src/App.js` — `<I18nProvider>` wraps the router

Missing Hindi/Gujarati keys fall back to English automatically.
